# Decision 0001: Harbor v0.1 Scope

Date: 2026-05-18

## Status

Accepted.

## Context

Harbor is intended to become a Better Auth-like ergonomic authentication
framework for Rust and Leptos applications. Authentication has a large security
surface, so v0.1 deliberately starts with a narrow email-based foundation.

The current implementation targets Leptos `0.8.x`, SQLx `0.8.x`, SQLite for
local development, and Resend for transactional email.

Source references:

- Leptos `0.8.19`: https://crates.io/crates/leptos/0.8.19
- Leptos server functions: https://book.leptos.dev/server/25_server_functions.html
- Leptos Axum extractors: https://book.leptos.dev/server/26_extractors.html
- SQLx `0.8.6`: https://crates.io/crates/sqlx/0.8.6
- Resend API introduction: https://www.resend.com/docs/api-reference/introduction

## Decision

Harbor v0.1 includes:

- email and password signup, signin, and signout;
- email signup confirmation;
- email OTP and magic-link signup/signin;
- email-based forgot password and password reset;
- server-side sessions in HttpOnly cookies;
- SQLx-backed storage with SQLite implemented first;
- Resend email delivery behind an explicit integration;
- Leptos components and server functions optimized for SSR, hydrate, CSR, and
  islands.

Harbor v0.1 excludes:

- OAuth and OpenID Connect;
- passkeys and WebAuthn;
- MFA;
- organizations, invitations, roles, and permissions;
- account linking;
- administrative dashboards;
- JWT-based browser sessions.

## Consequences

The v0.1 API can be small and difficult to misuse. Future features must extend
the existing typed foundation rather than bypassing it.

SQLite is the first concrete SQLx implementation, but the core store traits and
contract tests must remain database-portable. SQLx checked query macros are
database-specific, so future PostgreSQL and MySQL support should use separate
implementation modules and migrations. Cross-table operations that require
atomicity belong in the store trait contract rather than in web or service
code; every SQL driver must satisfy the same shared contract suite.

## Review Triggers

Review this decision before:

- adding a new auth provider;
- changing browser session storage;
- exposing SQLx types from `harbor-core`;
- treating email OTP or magic links as MFA;
- publishing Harbor crates.
